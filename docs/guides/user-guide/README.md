# Руководство пользователя (папка)

Единая точка входа для пользователя: основное руководство плюс визуальные
workflow, детализирующие отдельные процессы.

## Содержимое

- [[docs/guides/user-guide/USER_GUIDE|Основное руководство пользователя]] —
  сквозное «человеческое» объяснение: что нажать, что произойдёт и зачем.
- [[docs/guides/user-guide/workflows/README|Каталог workflow]] — пошаговые
  разборы процессов (кто исполняет, чем запускается, вход, результат, сигнал).

## Модель workflow-отчётов

Каждый процесс описывается двумя парными отчётами:

- **Полный процесс** — все исполнители (пользователь, CI, администратор).
- **Отчёты по ролям** — только действия одной роли, в логике «что сделать, чтобы
  добиться результата»: workflow пользователя и workflow администратора.

Каждый отчёт хранится как Markdown-страница (source of truth, попадает в граф
Obsidian) с приложенным самодостаточным HTML для наглядного просмотра в
`workflows/assets/`.
