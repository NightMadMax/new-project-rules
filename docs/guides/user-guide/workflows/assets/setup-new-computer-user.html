Один раз на каждой новой машине. Каждый шаг можно сделать двумя способами — попросить агента словами или запустить скрипт вручную. Результат один; выбирай, что удобнее.

Результат Компьютер готов: инструменты установлены, GitHub авторизован, общий vault открыт, глобальные правила агентов подключены и проверены.

Проверить и применить миграцию правил
Гдепапка Правила для нового проекта
🗣 агенту
«Построй план миграции глобальных правил и покажи; применяй только после моего подтверждения fingerprint»
⌨ вручную
План (ничего не меняет):./scripts/plan-migration.sh --plan --target global --report-onlyПрименить тем же fingerprint после review:./scripts/plan-migration.sh --apply --target global \ --fingerprint "<64-hex>" --yes
Готово когдаповторный --check показывает managed_match, backup сохранён
Безопасность: любой маршрут — агент сначала объясняет, что и зачем ставит/меняет, и ждёт разрешения. Setup не перезаписывает твои существующие правила: при конфликте показывает состояние и предлагает подтверждаемую миграцию.
Часть серии workflow пользователя. Детальный справочник: SETUP_NEW_COMPUTER, SYNC_GLOBAL_AGENTS, PLAN_MIGRATIONS.
