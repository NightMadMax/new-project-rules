Отправить на GitHub (открыть PR)
ГдеBest Practices— тот же агент
Что сделать · два маршрута
  1. 🗣 Агенту: «Запушь ветку и открой PR» — агент выполнит git push + gh pr create и спросит подтверждение перед созданием PR.
  2. ⌨ Вручную: git push -u origin candidate/…, затем gh pr create. Один PR — один кандидат.
Получишьоткрытый Pull Request; обоснование агента уходит в описание PR
⚙ самоGitHub Actions validate.yml проверяет PR. Красный чек блокирует merge — тогда исправить и запушить снова. Заходить никуда не надо.
⏳ ожиданиеАдминистратор разбирает кандидата (/review-practice-candidates): accept → практика попадает в practices/**, PR влит; reject → причина в кандидате. Целевой ответ — 7 дней.

Что сделать

  2. Написать: «Подтяни практики Best Practices под стек» · /apply-best-practices. Напоминание сработает само при определении стека.
