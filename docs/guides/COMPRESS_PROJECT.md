# Компрессия проекта

За долгую работу проект накапливает «мусор»: разросшиеся журналы, наслоившиеся
одноразовые research/audit, устаревшие промо-кандидаты, дрейф навигации,
технический мусор и раздутую память агентов. Компрессия периодически снимает эти
накопления, не теряя провенанс: записи не удаляются, а переносятся в архив.

Есть два уровня: детерминированный скрипт и скилл с суждением.

| Инструмент | Что делает | Безопасность |
|---|---|---|
| `compress-project.py` | Уровень 1: механическое, обратимое | отчёт по умолчанию |
| Скилл `compress-project` | Уровни 2–3: docs и память, с суждением | план перед применением |

## Скрипт (Уровень 1)

Read-only отчёт по умолчанию; `--apply` выполняет только обратимые операции.
Работает на любом стандартизированном проекте через `--root`; сам скрипт живёт в
репозитории правил и не копируется в проекты.

```sh
./scripts/compress-project.sh --root .
./scripts/compress-project.sh --root "/path/to/Project" --apply
```

Что делает по `--apply`:

- переносит записи `Fixed` старше `--fixed-max-age-days` (по умолчанию 90) из
  `docs/quality/DEFECTS.md` в `docs/quality/DEFECTS_ARCHIVE.md`; текст и сквозная
  нумерация сохраняются, разделы `Open`/`Won't Fix` и пояснения не трогаются;
- разрезает переросший `CHANGELOG.md` в `CHANGELOG_ARCHIVE.md`: триггер по
  размеру (`--changelog-max-kb`, по умолчанию 32), рез строго по границам версий,
  целевой размер `--changelog-target-kb` (24), всегда остаются `Unreleased` и
  последние `--changelog-keep` (5) релизов;
- удаляет регенерируемый мусор: `__pycache__`, `.DS_Store`, пустой `.trash`
  (каталог `.git` игнорируется).

Что только сообщает, не меняя (WARN/INFO):

- `last_verified` старше `--stale-days` (по умолчанию 60); плейсхолдеры
  незаполненные плейсхолдеры дат в угловых скобках и пустые значения, а также
  файлы под `templates/`, исключаются;
- размер `CHANGELOG.md`, когда он ниже порога.

Дрейф навигации (битые wikilinks, ссылки индексов на исчезнувшие файлы) ищет
`validate-project`; скрипт компрессии не дублирует эту логику.

## Скилл (Уровни 2–3)

Содержательную консолидацию, которую скрипт трогать не имеет права, ведёт скилл
`compress-project` с показом плана до применения:

- **Уровень 2** — слияние наслоившихся research/audit, чистка `applied`/`rejected`
  промо-кандидатов, починка дрейфа навигации.
- **Уровень 3** (`--include-memory`, по явному запросу) — память агентов вне git:
  строго отчёт → показ дублей → подтверждение → только слияние дублей и починка
  индекса `MEMORY.md`, без автоудаления файлов-фактов.

## Верификация

`python3 scripts/test-compress-project.py` — регрессионные тесты сплита, архива
дефектов, обнаружения мусора, staleness и идемпотентности. Матрица тестов:
[[docs/quality/TESTING|TESTING.md]].
