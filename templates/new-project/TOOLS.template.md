# Tools

Документируйте только нестандартные инструменты и локальные helper-скрипты.
Не дублируйте package manifests и стандартную справку CLI.

Сначала используйте существующий стек проекта. Предпочитайте стандартную
библиотеку и решения без новых зависимостей; для переносимой автоматизации
фиксируйте минимальную версию runtime.

Если отсутствующий инструмент заметно лучше доступной замены, сначала
зафиксируйте причину и запросите разрешение на установку. После установки
укажите package manager, версию и способ проверки.
